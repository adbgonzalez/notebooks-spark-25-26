# Comparativa entre motores de procesamento distribuído

## 1. MapReduce (Hadoop MapReduce)
![mapreduce](images/hadoop_mapreduce_logo.png)
### Descrición
Modelo de procesamento distribuído baseado en dúas fases ríxidas: **map → shuffle → reduce**. Foi o motor orixinal do ecosistema Hadoop.

### Características principais
- Procesamento **batch**
- Cada fase escribe e le datos en disco (**HDFS**)
- Modelo de programación pouco flexible
- Integrado de forma nativa con **Hadoop + YARN**

### Vantaxes
- Moi robusto e tolerante a fallos
- Escala ben en clústeres grandes
- Sinxelo conceptualmente (bo para explicar o paralelismo básico)

### Limitacións
- Moi lento para tarefas iterativas
- Verboso e pouco produtivo
- Non axeitado para streaming nin análises interactivas

### Cando usalo
- Contexto histórico
- Para entender os fundamentos do procesamento distribuído
- Casos batch moi simples e lineais

---

## 2. Apache Spark
![spark](images/apache_spark_logo.png)
### Descrición
Motor de procesamento distribuído **in-memory**, xeralista e unificado, que substitúe a MapReduce na maioría dos casos actuais.

### Características principais
- Procesamento **batch**, **streaming**, **SQL**, **ML** e **grafos**
- Execución en memoria (RDD, DataFrames)
- APIs en **Scala, Python, Java e SQL**
- Funciona sobre **YARN**, **Kubernetes** ou en modo *standalone*

### Vantaxes
- Moito máis rápido ca MapReduce
- Modelo flexible baseado en **DAG**
- Moi produtivo para desenvolvemento
- Estándar de facto en Big Data na actualidade

### Limitacións
- Maior consumo de memoria
- Curva de aprendizaxe maior ca MapReduce
- Precisa bo deseño para evitar *jobs* ineficientes

### Cando usalo
- Case sempre
- Procesamento analítico moderno
- Streaming e batch no mesmo motor
- Enxeñaría de datos e análise

---

## 3. Apache Flink
![flink](images/apache_flink_logo.png)
### Descrición
Motor deseñado **nativamente para streaming**, aínda que tamén soporta batch. Baseado en procesamento de eventos.

### Características principais
- Streaming real (**event-time**)
- Estado persistente e consistente
- Alta precisión temporal
- APIs en **Java/Scala** (Python máis limitada)

### Vantaxes
- Streaming máis potente e preciso ca Spark
- Baixa latencia
- Xestión avanzada de estado

### Limitacións
- Máis complexo
- Menos estendido en FP e contornos educativos
- Menos sinxelo para batch clásico

### Cando usalo
- Streaming complexo e crítico
- Procesamento orientado a eventos
- Casos onde a latencia é clave

---

## 4. Apache Storm
![storm](images/apache_storm_logo.png)
### Descrición
Un dos primeiros motores de **streaming**, hoxe en día bastante substituído por Flink e Spark Streaming.

### Características principais
- Streaming en tempo real
- Topoloxías baseadas en **spouts** e **bolts**
- Moi baixo nivel

### Vantaxes
- Latencia moi baixa
- Control fino do fluxo

### Limitacións
- Programación complexa
- Non soporta batch
- Practicamente obsoleto

### Cando usalo
- Casos moi específicos
- Contexto histórico

---

## 5. Dask
![dask](images/dask_logo.png)
### Descrición
Motor de computación distribuída orientado ao ecosistema **Python**.

### Características principais
- Paralelismo en Python
- APIs similares a **pandas** e **numpy**
- Execución en clúster ou en local

### Vantaxes
- Moi integrado co ecosistema Python científico
- Sinxelo para usuarios de pandas
- Bo para *workloads* medianos

### Limitacións
- Non pensado para Big Data “clásico”
- Escalabilidade menor ca Spark ou Flink
- Non estándar en contornos empresariais

### Cando usalo
- Ciencia de datos en Python
- Escenarios onde Spark é excesivo

---

## 6. Resumo comparativo

| Motor       | Batch | Streaming | Velocidade | Facilidade | Uso actual |
|------------|-------|-----------|------------|------------|------------|
| MapReduce  | si    | non       | baixa      | baixa      | histórico  |
| Spark      | si    | si        | alta       | alta       | estándar   |
| Flink      | si    | si        | moi alta   | media      | crecente   |
| Storm      | non   | si        | alta       | baixa      | residual   |
| Dask       | si    | limitado  | media      | alta       | nicho      |

> **MapReduce** ensina como funciona o procesamento distribuído.
